# OpenClaw Team OS UI Redesign Plan

## 1. 改版目标

本轮不是“换皮”，而是把当前客户端从工程感很强的多卡片后台，重构成更像正式创作客户端的工作台。

参考方向来自 `Holopix` 首页的几个优点：

- 左侧导航是清晰的功能入口，不只是图标竖排。
- 顶部有持续存在的操作条，用户随时知道“现在能做什么”。
- 中央主工作区有明显主次，不会把所有内容都做成同级卡片。
- 视觉上是深色、沉浸、工具化，而不是普通 SaaS 管理后台。
- 每个页面都有一个明确主任务，而不是“所有信息都塞进来”。

本项目不会照搬 Holopix 的页面或代码，只吸收它的结构逻辑和工具气质。

## 2. 当前 UI 问题清单

本轮确认需要解决的 UI / 交互问题共 **16 项**：

1. 左侧导航过窄，只有图标，没有功能语义。
2. 顶部没有稳定操作条，用户进入后缺少“当前上下文”。
3. 页面头部高度不稳定，很多页面浪费首屏空间。
4. 入口页信息稀疏，大面积留白，但真正的进入动作不够突出。
5. 全站大量卡片样式过于相似，缺少主次层级。
6. 招聘页文字密度高，方向、筛选、招聘对象之间层级不清。
7. 员工页和团队页为了铺满而铺满，信息密度低、空间利用差。
8. 任务台同时承载输入、结果、审批提示，结构混乱。
9. 短剧流程虽然已经阶段化，但“职能线切换”仍然不够醒目。
10. 短剧结果审查区太小，不适合视频/分镜审片。
11. 审批页和设置页仍然偏后台表单气质，不像生产工具。
12. 当前页面缺少统一的“侧边状态摘要”，用户不知道当前组织处于什么状态。
13. 页面之间缺少一致的视觉骨架，导致切页像进入不同产品。
14. 主按钮和次按钮层级不够稳定，很多页面第一操作不够明显。
15. 卡片文案普遍偏长，影响扫描效率。
16. 自动化测试主要覆盖功能链路，但对布局重构缺少明确页面验收标准。

## 3. 本轮优化范围

本轮优化范围覆盖 **8 个页面 / 视图**：

1. 客户端进入页
2. 组织总览
3. 招聘
4. 员工
5. 团队
6. 任务台
7. 审批
8. 设置

另加 **1 套全局骨架改造**：

- 全局左侧导航
- 顶部操作条
- 页面标题区
- 通用卡片体系
- 全局按钮与筛选器层级

## 4. 新的全局结构

新的客户端结构统一为：

1. 左侧：宽导航栏  
   展示品牌、当前组织、主要功能入口、状态摘要。

2. 顶部：操作条  
   展示当前页面上下文、快速提示、角色、当前单位、重点动作。

3. 中央：主工作区  
   每个页面只保留 1 个主任务区和 1-2 个辅助区。

4. 右侧：不再全局固定  
   改成按页面决定是否需要审查台/详情面板，避免机械三栏。

## 5. 页面级改造策略

### 5.1 客户端进入页

- 删除无意义说明条目。
- 保留“进入客户端”这一个核心动作。
- 左侧强调品牌和工作台属性，右侧做接入方式切换。
- 整体更像“启动台”，而不是 marketing 落地页。

### 5.2 组织总览

- 从四块平铺指标卡，改成“组织状态 + 下一步建议”。
- 把推荐招聘、工作焦点做成更明确的主次。
- 首屏减少泛描述，增强行动导向。

### 5.3 招聘

- 强化方向分类卡的视觉识别。
- 员工 / 团队目录卡片加入更强的图标与节奏信息。
- 缩短招聘按钮文案，避免大块 CTA 压满卡片底部。
- 招聘页优先服务“快速找到该招谁”，而不是解释系统。

### 5.4 员工 / 团队

- 统一成“目录 + 档案”的客户端布局。
- 左侧列表用于切换单位，右侧集中展示名称、技能、职责、交付能力、最近活动。
- 避免用巨大空白把内容稀释。
- 支持直接重命名单位。

### 5.5 任务台

- 普通任务保持“输入 → 最近任务 → 结果复用”。
- 短剧任务台升级为更明显的项目流：
  - 职能线切换
  - 阶段推进
  - 阶段主工作区
  - 大尺寸阶段审查台
- 不再让结果区缩在右下角做小卡片审查。

### 5.6 审批

- 审批卡减少说明性废话，突出摘要、风险、动作。
- 审批页面强调“快速判断”，不是阅读长文。

### 5.7 设置

- 继续保留低频管理能力，但改成更整洁的配置区。
- 减少“后台系统感”，更像客户端里的组织控制面板。

## 6. 短剧工作流的 UI 收口

短剧工作流本轮继续遵守这条顺序：

1. 剧情立项
2. 人物定稿
3. 剧本分镜
4. 视频制作
5. 审片导出

其中角色分工明确为：

- `短剧编导员工`
  负责剧情钩子、人物关系、对白、分镜拆解、转场节奏。

- `AI 短剧制作团队`
  负责人物一致性、镜头包装、出片接力、视频生成与审片收口。

UI 上的目标是让用户始终知道：

- 我现在在哪个阶段
- 当前应该由哪条职能线接手
- 我现在该审的是人物、分镜，还是视频样片

## 7. 自动化验证要求

本轮 UI 重构后，至少执行：

- `pnpm build`
- `pnpm e2e:smoke`

如果桌面端脚本与新布局存在偏差，需要同步修复后再跑：

- `pnpm e2e:desktop-smoke`

## 8. 本轮交付标准

本轮完成后，客户端应满足：

- 打开后整体气质明显接近“专业创作工作台”
- 左侧导航不再是无语义图标栏
- 全部主页面都有清晰的主任务区
- 员工 / 团队 / 任务台的信息密度更合理
- 短剧任务流的阶段和审查关系一眼能看懂
- 自动化构建和基础烟测通过
