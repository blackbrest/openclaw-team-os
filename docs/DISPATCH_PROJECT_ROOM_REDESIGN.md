# 项目内工作台 Discord 化重构需求

## 1. 背景

当前“任务台 -> 项目内页面”仍然沿用后台信息卡片式布局，信息虽然全，但缺少即时协作工具的沉浸感。

本轮目标不是继续堆卡片，而是把项目内页面重构成更接近 Discord 的项目协作界面：

- 左侧是清晰的项目成员入口
- 中间是主要对话区
- 右侧是项目进度预览与后续扩展位
- 项目上下文只围绕当前项目，不再反复展示“任务台”页面级标题

## 2. 本轮范围

本轮只收口“任务台”中的两个层级：

1. 任务台全局左侧导航中的项目入口
2. 进入某个项目后的项目内协作页面

不改动招聘、员工、审批、设置等其他主页面的信息架构。

## 3. 核心需求

### 3.1 全局左侧导航新增项目入口区

- 在最左侧全局导航中，“任务台”入口下方需要展示已创建的全部项目。
- 新项目创建成功后，左侧项目入口需要立即同步出现。
- 用户在任意页面都可以通过左侧项目入口直接进入对应项目。
- 项目入口需要高亮当前打开的项目，降低迷路感。

### 3.2 项目内页面去掉“任务台”大标题

- 进入项目后，页面顶部不再显示通用的“任务台”页头。
- 顶部只保留当前项目名称、项目描述与必要操作按钮。
- 项目内整个主区域都应该服务当前项目，不再保留无关页面说明。

### 3.3 项目内采用 Discord 式三栏结构

- 左栏：项目成员列表
  - 展示当前项目内已招聘员工
  - 视觉宽度保持紧凑，不要过宽
  - 每项只需要优先展示头像与员工名
  - 项目总群、主管汇总、专项群组也应有可进入入口，但不能抢成员列表主位

- 中栏：会话主区
  - 作为当前项目的核心操作区
  - 主要展示消息流与输入框
  - 支持给当前选中对象发送消息
  - 增加键盘交互：`Enter` 发送，`Shift + Enter` 换行

- 右栏：项目进度预览
  - 当前阶段、主管汇总、关键进度指标放在这里
  - 短剧项目流水线、最近产出、主管派工、员工回报等相关能力优先归到右栏
  - 右栏先作为“项目进度与协作控制面板”，后续扩展能力继续放这里

### 3.4 项目设置入口

- 每个项目页面右上角增加“项目设置”按钮。
- 项目名称、项目描述、项目主管等设置统一从这里进入。
- 设置入口应始终和当前项目绑定，避免散落到其他页面。

## 4. 交互要求

- 点击左侧项目入口，直接进入该项目
- 点击项目成员，切换到对应会话
- 点击项目总群/主管汇总/专项群组，切换到对应频道
- 消息输入框支持：
  - `Enter` 直接发送
  - `Shift + Enter` 保留换行
- 创建项目后：
  - 自动进入新项目
  - 左侧全局导航同步出现该项目入口

## 5. 视觉方向

- 参考 Discord 的“深色、沉浸、低装饰、强结构”气质
- 避免继续使用大面积后台卡片拼贴
- 通过栏位、分割、头像、列表状态和消息流层级建立秩序
- 左栏与右栏都应弱于中间会话区，中间会话区必须是视觉主角

## 6. 非目标

- 本轮不做真正的多用户在线同步
- 本轮不重做后端项目模型
- 本轮不新增复杂权限系统
- 本轮不把所有旧功能都做成独立新页面，优先完成布局重构与关键入口收口

## 7. 验收标准

- 左侧全局导航能看到全部项目入口
- 新建项目后左侧项目入口立即同步出现
- 进入项目后不再看到“任务台”页头，而是项目名称和描述
- 项目内形成清晰的左成员 / 中会话 / 右进度三栏结构
- 消息输入支持回车发送
- 项目右上角有明确设置入口
- `pnpm build` 通过
